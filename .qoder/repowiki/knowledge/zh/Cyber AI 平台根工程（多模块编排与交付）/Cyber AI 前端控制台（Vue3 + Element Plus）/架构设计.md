单页应用，入口 `src/main.js` 挂载 App、Pinia、vue-router、Element Plus 并全局注册所有图标；`vite.config.js` 通过 dev server 将 `/api` 反向代理到后端 `http://localhost:8080`，并对 SSE (`text/event-stream`) 请求额外设置 `no-cache`、移除 `content-length`、关闭 `x-accel-buffering` 以支持流式响应。

分层结构：
- `router/index.js` 集中声明路由与 `requiresAuth` 守卫，按功能划分 Home/Chat/Memory/RAG/Agent/SearchAgent/MultiAgent/Structured/Tools/Mcp 十个视图；
- `stores/user.js`、`stores/chat.js` 使用 Pinia 管理登录态与会话列表；
- `utils/request.js` 封装 Axios 实例，统一处理 Bearer Token 注入、401 自动刷新（带并发订阅队列）、SSE 连接（`createSSEConnection` 基于 `fetch` + `ReadableStream` 解析 `data:` 事件），并提供 `get/post/put/del` 快捷方法；
- `api/index.js` 面向业务聚合 API，对 SSE 接口封装为返回 Promise 且附带 `.close()` 的流式调用（如 `streamChat`、`multiAgentChat`），同时暴露 RESTful 会话、RAG 文档、MCP 工具等函数；
- `views/*` 为页面级组件，`components/ChatInput.vue`、`components/ChatMessage.vue` 为可复用 UI 片段；
- `assets/styles/global.css` 提供 Markdown 渲染样式，`App.vue` 内嵌代码高亮主题与 markdown-body 样式。

依赖方向：views → api → utils/request → axios/fetch；router/stores 被 App 与视图共同消费，无循环引用。
