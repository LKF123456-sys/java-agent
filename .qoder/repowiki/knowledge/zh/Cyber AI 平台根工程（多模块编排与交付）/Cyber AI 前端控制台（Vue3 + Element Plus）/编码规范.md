- API 层统一通过 `@/utils/request` 导出的 `get/post/put/del` 发起 HTTP 请求，避免直接 import axios。
- 需要鉴权的视图路由在 `meta.requiresAuth = true` 中声明，由 router 全局前置守卫根据 `userStore.isLoggedIn` 拦截跳转。
- 流式 SSE 接口一律使用 `createSSEConnection` 建立连接，并通过 `{ onMessage, onError, onClose }` 回调驱动，错误通过 `[ERROR]` 前缀消息触发 reject。
- Token 刷新采用 `isRefreshing` 标志位 + `refreshSubscribers` 队列模式，保证并发 401 请求只触发一次刷新并串行重试。
- 视图组件遵循 `<template>/<script setup>/<style scoped>` 三段式 SFC 结构，状态集中在 Pinia store 或组件 ref/computed 中。
