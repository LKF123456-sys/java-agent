模块采用 Spring Boot 单体分层架构，入口为 `AiLearnApplication`（排除 PgVector 自动配置以避免 Bean 冲突），按功能域拆分为多个子包：
- `common`：统一响应 `Result<T>`、错误码 `ErrorCode`、业务异常 `BusinessException` 与全局异常处理 `GlobalExceptionHandler`，所有 Controller/Filter 的错误均收敛至此。
- `security`：无状态安全体系——`SecurityConfig` 禁用 CSRF/表单登录，注册 `JwtAuthenticationFilter`（继承 `OncePerRequestFilter`，支持 ASYNC/FORWARD 恢复 SecurityContext 以适配 SSE 流式响应），配合 `JwtUtil` 与 `UserPrincipal` 完成双 Token（access/refresh）鉴权。
- `config`：横切配置集中管理，包括 `WebConfig`（CORS、静态资源映射、SPA 路由转发）、`SecurityConfig`、`MyBatisPlusConfig`、`RateLimiterConfig`（Resilience4j 令牌桶实例 per-service）、`OpenApiConfig`、`McpServerConfig`、`SpaFallbackController/Filter` 以及 MDC 链路追踪过滤器 `MdcTraceFilter`。
- 业务层按领域划分：`chat`（基础对话）、`memory`（带持久化记忆的对话，使用 `DatabaseChatMemory`）、`agent` / `multi-agent` / `search-agent`（单 Agent 工具调用与 Planner/Researcher/Coder/Critic/Executor 多 Agent 协作）、`rag`（文档上传→向量存储→检索增强问答）、`structured`（从非结构化文本提取结构化信息）、`tools`（Calculator/Weather/WebSearch 可注册工具）、`mcp`（Model Context Protocol 暴露系统工具）。
- 数据访问层遵循 MyBatis-Plus 约定：`entity` 定义表结构，`mapper` 接口由 `@MapperScan("com.ailearn.mapper")` 扫描，SQL 初始化通过 `spring.sql.init.schema-locations` 在默认 profile 加载 `schema.sql`、docker profile 切换为 `schema-postgresql.sql`；PostgreSQL + PgVector 仅在 docker profile 下启用。
- 前端 SPA 静态资源打包在 `resources/static/assets`，由 `WebConfig` 和 `SpaFallbackController` 将 `/chat`、`/rag`、`/agent` 等前端路由转发到 `index.html`。
- 依赖方向：Controller → Service → Mapper/外部 SDK（Ollama/Tavily/PgVector），Security/Config/Common 作为横切层被各层注入，无反向依赖。
