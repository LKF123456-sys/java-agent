本模块由两部分组成：
- `docker/app/Dockerfile`：采用两阶段构建，第一阶段基于 `maven:3.9.9-eclipse-temurin-21` 执行 `mvn dependency:go-offline` 预拉依赖并 `mvn clean package -DskipTests` 产出 `cyber-ai-platform.jar`；第二阶段基于 `eclipse-temurin:21-jre` 仅复制 JAR 及 `/app/logs`、`/app/uploads` 目录，通过 `JAVA_OPTS` 注入 JVM 参数并以 `ENTRYPOINT ["sh","-c",...]` 启动。
- `docker/logstash/`：Logstash 运行时配置，`config/logstash.yml` 指定 HTTP API 监听地址、pipeline 工作线程与批量大小；`pipeline/logstash.conf` 定义 input→filter→output 三段式管道，从 `/app/logs/app-json-*.log` 读取 JSON 日志，用 `date` 过滤器将 `timestamp` 映射为 `@timestamp`，重命名 `message` 避免冲突，最终写入按日期分片的 `cyber-ai-logs-%{+YYYY.MM.dd}` Elasticsearch 索引，同时保留 `stdout` 调试输出。
- `.github/workflows/ci-cd.yml`：三 job 并行流水线——`backend-build-test`（JDK 21 + Maven verify，仅在 main/master 分支触发 Docker 构建与 Trivy 漏洞扫描）、`frontend-build`（Node 20，在 `./frontend` 子目录执行 `npm ci && npm run build` 产出 `frontend/dist/` 制品）、`code-quality`（占位检查）。后端镜像以 `${{ github.sha }}` 打标签，测试报告上传至 `test-results` artifact。
