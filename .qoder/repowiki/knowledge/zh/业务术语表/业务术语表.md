### RAG
- 定义：检索增强生成（Retrieval-Augmented Generation），本项目指将用户上传的 PDF/Word/文本等文档解析切分后向量化存入 PgVector，问答时先检索相似片段再注入系统提示词供大模型参考回答的模式。
- 别名：知识库问答

### Planner Agent
- 定义：多 Agent 协作流水线中的规划角色，负责分析用户任务复杂度并输出 JSON 路由决策（complexity/needResearch/needCoder），决定后续是否执行 Researcher/Coder。
- 别名：规划Agent

### Researcher Agent
- 定义：多 Agent 协作中的信息收集角色，具备工具调用能力（天气、搜索、计算等），用于在执行前获取实时信息辅助决策。
- 别名：研究Agent

### Coder Agent
- 定义：多 Agent 协作中的代码生成角色，接收编程任务描述并输出完整代码及解释，可与 Critic 迭代循环以提升质量。
- 别名：编码Agent

### Critic Agent
- 定义：多 Agent 协作中的质量审查角色，对 Coder 产出的代码进行审查，输出「审查结果：通过」或修改意见，支持最多 MAX_CRITIC_ITERATIONS 轮迭代直至通过或达到上限。
- 别名：审查Agent

### Executor Agent
- 定义：多 Agent 协作的最终整合角色，汇总 Planner/Researcher/Coder/Critic 各阶段输出，给出面向用户的最终答案。
- 别名：执行Agent

### 赛博搜索官
- 定义：SearchAgentService 中内置的系统提示词所定义的联网搜索助手人格，遵循「先搜索后总结」原则，要求回答中标注来源链接。
- 别名：搜索Agent

### SSE 事件流
- 定义：多 Agent 协作采用 Server-Sent Events 推送 token 级流式响应，每个事件为 JSON 对象，type 字段取值包括 agent_start/token/agent_end/info/error/done/init，agent 字段标识当前活跃角色。
- 别名：SSE事件

### 会话记忆隔离
- 定义：不同 Agent 通过给 conversationId 添加前缀（planner_/researcher_/coder_/critic_/executor_/agent_/search_/travel_/task_）来隔离各自的历史上下文，避免跨角色记忆污染。
- 别名：记忆隔离
